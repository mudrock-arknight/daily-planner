# 个人规划助手 v2 - 产品需求文档

## Overview
- **Summary**: 为个人规划应用添加用户自主完成标记功能，完善统计系统，优化计划时间排布，并进行全面的前端界面升级
- **Purpose**: 让用户能够主动标记任务完成状态，提供准确的完成率统计，优化时间块排布让计划更紧凑，提供更美观现代的用户界面
- **Target Users**: 需要详细规划日常学习的大学生

## Why
当前应用存在以下问题需要解决：
1. 时间块"已完成"状态由系统自动判断，用户无法手动标记
2. 缺少用户自定义的完成情况统计
3. 选择其他天的计划后会跳回当天
4. 计划时间存在间隔，没有充分利用时间
5. 界面设计较为普通，缺乏独特的视觉风格

## What Changes

### 核心功能改动
1. **用户完成标记功能** - 用户可点击时间块手动标记完成/未完成
2. **完成情况统计系统** - 后台记录并汇总完成数据到统计页面
3. **日期选择修复** - 修复切换日期后自动跳回当天的问题
4. **时间块紧凑排布** - 确保计划时间无间隔，充分利用每一天
5. **前端界面全面优化** - 使用 frontend-design skill 提升视觉设计

### 数据模型改动
- 新增 `timeblock_completions` 表存储用户完成标记
- 时间块数据结构增加 `completed` 和 `completedAt` 字段

## Impact
- **Affected specs**: 所有页面（HomePage, WeeklyPlanPage, StatsPage）
- **Affected code**: 
  - 新增 Supabase 表结构
  - HomePage 和 WeeklyPlanPage 增加完成标记交互
  - StatsPage 增加完成率统计图表
  - 全局样式和主题优化

## ADDED Requirements

### Requirement: 时间块完成标记
系统 SHALL 提供用户自主标记时间块完成状态的功能

#### Scenario: 用户标记完成
- **GIVEN**: 用户在首页或周计划页查看时间块
- **WHEN**: 用户点击时间块的完成按钮
- **THEN**: 时间块显示为已完成状态，后台记录完成时间

#### Scenario: 用户取消完成
- **GIVEN**: 时间块已标记为完成
- **WHEN**: 用户再次点击完成按钮
- **THEN**: 时间块恢复为未完成状态，删除完成记录

### Requirement: 完成情况统计
统计页面 SHALL 展示用户的时间块完成情况汇总

#### Scenario: 查看完成率
- **GIVEN**: 用户访问统计页面
- **WHEN**: 查看统计数据
- **THEN**: 显示本周/本月的完成率、完成数量统计、趋势图表

### Requirement: 日期选择稳定性
切换日期后 SHALL 保持在该日期，不自动跳回当天

#### Scenario: 切换到其他日期
- **GIVEN**: 用户在首页或周计划页
- **WHEN**: 用户点击选择其他日期
- **THEN**: 页面应停留在用户选择的日期，直到用户主动切换

### Requirement: 时间块紧凑排布
计划生成 SHALL 确保时间块之间无间隔，充分利用每一天的时间

#### Scenario: 生成日计划
- **GIVEN**: 生成新的一天计划
- **WHEN**: 安排时间块
- **THEN**: 8:00-23:00 之间应尽可能排满，只保留必要的吃饭、休息时间

## MODIFIED Requirements

### Requirement: 统计页面增强
原统计页面仅显示打卡数据，现需要增加完成率统计

- **原有功能**: 打卡天数、累计学习时长、学习趋势
- **新增功能**: 
  - 本周时间块完成率（已完成/总时间块）
  - 各类型任务完成率对比（课程、学习、休息等）
  - 每日完成率柱状图

## REMOVED Requirements
- 无

## Acceptance Criteria

### AC-1: 时间块完成标记
- **Given**: 用户查看时间块列表
- **When**: 点击时间块的圆形勾选按钮
- **Then**: 
  - 按钮变为绿色填充状态，显示勾选图标
  - 按钮旁边文字变为"已完成"
  - 后台记录完成状态到 Supabase
- **Verification**: `human-judgment` + `database-check`

### AC-2: 完成率统计展示
- **Given**: 用户打开统计页面
- **When**: 查看统计数据
- **Then**: 
  - 显示本周完成率百分比和完成数量
  - 显示各类型任务完成率对比图
  - 显示每日完成率趋势图
- **Verification**: `human-judgment`

### AC-3: 日期选择稳定
- **Given**: 用户在首页
- **When**: 选择周一至周日中的任意一天
- **Then**: 页面停留在选择的日期，不会自动跳转回当天
- **Verification**: `human-judgment`

### AC-4: 时间块紧凑排布
- **Given**: 查看生成的日计划
- **When**: 检查时间安排
- **Then**: 
  - 8:00-23:00 之间各时间段紧密衔接
  - 每天至少有 12 个小时的有效时间安排
  - 休息时间仅保留吃饭、洗漱必要时间
- **Verification**: `human-judgment`

### AC-5: 界面设计优化
- **Given**: 用户打开应用
- **When**: 查看整体界面
- **Then**: 
  - 界面具有独特、难忘的视觉风格
  - 动画流畅自然，符合直觉
  - 配色方案统一且有层次感
  - 排版舒适，留白恰当
- **Verification**: `human-judgment`
